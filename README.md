# knockback-fighter

3D の吹っ飛ばし系対戦アクションゲームを学習しながら試作する Unity プロジェクトです。  
既存IPの再現ではなく、**ゲーム開発の基礎を学ぶためのオリジナル試作**として進めます。

## 目的

このプロジェクトでは、Unity を使って以下を学ぶことを目的とします。

- Unity 6 / Universal 3D による 3D ゲーム開発
- Input System を使ったキーボード・コントローラー入力
- Rigidbody ベースの移動とジャンプ
- 当たり判定、吹っ飛び、落下、リスポーンなどの基礎実装
- Cursor を使ったバイブコーディングによる高速試作

## 最初の完成目標

まずは以下を満たす最小構成を目指します。

- 3D ステージが 1 つある
- プレイヤーキャラクターが 1 体いる
- プレイヤーが移動できる
- プレイヤーがジャンプできる
- 通常攻撃を出せる
- 攻撃が相手に当たる
- 相手が吹っ飛ぶ
- ステージ外に落ちたらリスポーンする

## スコープ外

以下は当面後回しにします。

- オンライン対戦
- 複数キャラクター差分
- アイテム
- ステージギミック
- 高度なコンボシステム
- 本格的な演出やUIの作り込み
- 既存IPの厳密な再現

## 開発方針

- まずは **動くものを最優先** で作る
- 見た目よりも **入力 / 移動 / ジャンプ / 攻撃 / 吹っ飛び** を優先する
- プレイヤー本体のロジックと見た目は分離する
- 問題が起きたら、複雑化する前に最小構成で切り分ける
- Cursor には README の方針を前提に修正させる

## 既知の問題

- 攻撃 / 被弾 / ノックバックは最小実装のため、当たり判定のサイズ・発生時間の継続調整が必要
- ペンギン翼モーションと当たり判定の見た目一致はまだ微調整が必要
- 攻撃踏み込み量（距離・時間）は調整余地あり
- 現状のキャラクター選択はペンギンのみ（プレイヤー/CPUともに固定）
- シーン側の SerializeField 値が古いと挙動がズレるため、シーン保存・コミット運用を徹底する必要がある

## 次にやること

優先順位は以下の通りです。

1. 当たり判定（Hitbox/Hurtbox）のサイズと発生時間を継続調整する
2. 移動のバリエーションを追加（歩く、ダッシュ）
   - スマブラ風に、弾き入力でダッシュできるようにする
3. 空中ジャンプを追加（空中で1回だけジャンプ可能）
   - 将来的にキャラごとの複数回ジャンプへ拡張可能な設計にする
4. タイトル画面を追加する
   - フローを「タイトル → キャラクター選択 → バトル開始（3,2,1カウントダウン）」にする
5. キャラクター選択を複数キャラ対応に拡張する
   - プレイヤー/CPUそれぞれで選択可能にする
6. CPU対戦を実装する（最小AI）
   - 接近→攻撃→復帰の基本行動で 1P vs CPU を成立させる
7. ダミー敵をプレハブ化してシーン常設化する
8. UI の最小実装（残機/状態表示など）を追加する
