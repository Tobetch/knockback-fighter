# knockback-fighter

3D の吹っ飛ばし系対戦アクションゲームを学習しながら試作する Unity プロジェクトです。  
既存IPの再現ではなく、**ゲーム開発の基礎を学ぶためのオリジナル試作**として進めます。

## 目的

このプロジェクトでは、Unity を使って以下を学ぶことを目的とします。

- Unity 6 / Universal 3D による 3D ゲーム開発
- Input System を使ったキーボード・コントローラー入力
- Rigidbody ベースの移動とジャンプ
- 当たり判定、吹っ飛び、落下、リスポーンなどの基礎実装
- Cursor を使ったバイブコーディングによる高速試作

## 最初の完成目標

まずは以下を満たす最小構成を目指します。

- 3D ステージが 1 つある
- プレイヤーキャラクターが 1 体いる
- プレイヤーが移動できる
- プレイヤーがジャンプできる
- 通常攻撃を出せる
- 攻撃が相手に当たる
- 相手が吹っ飛ぶ
- ステージ外に落ちたらリスポーンする

## スコープ外

以下は当面後回しにします。

- オンライン対戦
- 複数キャラクター差分
- アイテム
- ステージギミック
- 高度なコンボシステム
- 本格的な演出やUIの作り込み
- 既存IPの厳密な再現

## 開発方針

- まずは **動くものを最優先** で作る
- 見た目よりも **入力 / 移動 / ジャンプ / 攻撃 / 吹っ飛び** を優先する
- プレイヤー本体のロジックと見た目は分離する
- 問題が起きたら、複雑化する前に最小構成で切り分ける
- Cursor には README の方針を前提に修正させる

## 既知の問題

- カメラのサイドビュー角度がまだ要調整（真横固定のチューニング中）
- 攻撃 / 被弾 / ノックバックは最小実装のため、当たり判定のサイズ調整が必要
- シーン側の SerializeField 値が古いと挙動がズレるため、シーン保存・コミット運用を徹底する必要がある

## 次にやること

優先順位は以下の通りです。

1. カメラをサイドビュー（真横）として安定させる
2. ダミー敵を追加して、攻撃→被弾→ノックバックを実機検証する
3. 落下→リスポーン後の挙動を調整する
4. 当たり判定（Hitbox/Hurtbox）のサイズと発生時間を調整する
5. 最小 UI を追加する
